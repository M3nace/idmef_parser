Aggregates

Node (1..1)

A host or network involved in the incident. See Section 3.18.

NodeRole (0..*)

The intended purpose of the system. See Section 3.18.2.

Service (0..*)

A network service running on the system. See Section 3.20.

OperatingSystem (0..*)

The operating system running on the system.

Counter (0..*)

A counter with which to summarize properties of this host or network. See Section 3.18.3.

AssetID (0..*)

An asset identifier for the System.

Description (0..*)

A free-form text description of the System.

AdditionalData (0..*)

A mechanism by which to extend the data model.

category (0..1)

Classifies the role the host or network played in the incident. These values are maintained in the "System- category" IANA registry per Section 10.2.

ext-category (0..1)

A means by which to extend the category attribute. See Section 5.1.1.

interface (0..1)

Specifies the interface on which the event(s) on this System originated. If the Node class specifies a network rather than a host, this attribute has no meaning.

spoofed (0..1)

An indication of confidence in whether this System was the true target or attacking host. The permitted values for this attribute are shown below. The default value is "unknown".

virtual (0..1)

Indicates whether this System is a virtual or physical device. The default value is "unknown".

ownership (0..1)

Describes the ownership of this System relative to the victim in the incident. These values are maintained in the "System-ownership" IANA registry per Section 10.2.

ext-ownership (0..1)

A means by which to extend the ownership attribute. See Section 5.1.1.

restriction (0..1)

See Section 3.3.1.

ext-restriction (0..1)

A means by which to extend the restriction attribute. See Section 5.1.1.

observable-id (0..1)

See Section 3.3.2.
